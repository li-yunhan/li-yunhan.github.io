Spring

一 Spring概述

1.1 概述

Spring是一个IOC(DI)和AOP容器框架。

IOC就理解为一个容器，放置一些Bean，也就是一些资源信息。

1.2 Spring的优良特性

  1. 非侵入式：基于Spring开发的应用不是一定要调用Spring中的API
  2. 依赖注入：DI——Dependency Injection，反转控制(IOC)最经典的实现。
  3. 面向切面编程：Aspect Oriented Programming——AOP
  4. 容器：Spring是一个容器，因为它包含并且管理应用对象的生命周期
  5. 组件化：Spring实现了使用简单的组件配置组合成一个复杂的应用。在 Spring 中可以使用XML和Java注解组合这些对象。

第二章 IOC容器和Bean的配置

2.1 IOC和DI

  1. IOC(Inversion of Contrl)：反转控制 应用用程序中的组件想要获取资源时，在传统方式中，需要组件主动的从容器中获取资源。反转控制的思想就是，容器主动的把资源推送给需要的组件，开发人员并不需要知道容器是怎么创建资源对象的 把对象创建和对象之间的调用过程，交给Spring进行管理 这样做可以降低耦合度 实现方法：通过xml文件，工厂模式和反射实现的
  2. DI(Dependency Injection)：依赖注入 IOC的另一种表述方式：即组件以一些预先定义好的方式(例如：setter 方法)接受来自于容器的资源注入。
    IOC 描述的是一种思想，而DI 是对IOC思想的具体实现。

  2. ApplicationContext 是BeanFactory的子接口，基本上实例化IOC容器都是通过这个方法

2.4 两种Bean类型

Spring 有两种类型 bean，一种普通 bean，另外一种工厂 bean（FactoryBean）

  1. 普通 bean：在配置文件中定义 bean 类型就是返回类型
  2. 工厂 bean：在配置文件定义 bean 类型可以和返回类型不一样
    1. 创建类，让这个类作为工厂 bean，实现接口 FactoryBean
    2. 实现接口里面的方法，在实现的方法中定义返回的 bean 类型

2.5 Bean的作用域

可以在元素的scope属性里设置bean的作用域，以决定这个bean是单实例的还是多实例的。就是决定了Bean在什么时候会有效果。

2.6 Bean的生命周期

2.6.1 生命周期

IOC容器就是对Bean进行管理，Spring允许在Bean生命周期内特定的时间点执行指定的任务，生命周期如下：

  1. 通过构造器或工厂方法创建Bean实例
  2. 为Bean的属性设置值和对其他Bean的引用（Set方法）
  3. 调用Bean的初始化方法（这个方法是自己写的）
  4. 使用Bean
  5. 当容器关闭时，调用Bean的销毁方法

在配置bean时，通过init-method和destroy-method 属性为bean指定初始化和销毁方法。

2.6.2 后置处理器

Bean后置处理器允许在调用初始化方法前后对Bean进行额外的处理，是对IOC容器中的所有Bean实例逐一处理的。
典型应用就是对Bean的属性值进行判断。
Bean后置处理器需要实现接口org.springframework.beans.factory.config.BeanPostProcessor，并重写两个方法。

2.7 引用外部属性文件

当bean的配置信息逐渐增多时，查找和修改一些bean的配置信息就变得愈加困难。这时可以将一部分信息提取到bean配置文件的外部。

  1. 创建外部属性文件，properties 格式文件，写数据库信息（jdbc.properties）
  2. 把外部 properties 属性文件引入到 spring 配置文件中 —— 引入 context 名称空间

2.8 使用注解配置Bean

2.8.1 组件说明

在XML中配置Bean太麻烦，使用注解的方法会显得Bean更加简洁和优雅。

  1. @Component：标识受IOC容器管理的组件
  2. @Repository：标识一个受IOC容器管理的持久化层组件（DAO层）
  3. @Service：业务逻辑层组件
  4. @Controller：表述层控制组件

这几个组件的效果都是为IOC容器中添加组件，不同的命名也只是为了帮助开发者区分而已。

第四章 AOP概述

4.1 AOP概述

  1. 面向切面编程（方面），利用 AOP 可以对业务逻辑的各个部分进行隔离，从而使得 业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。
  2. 通俗描述：不通过修改源代码方式，在主干功能里面添加新功能
  3. 使用登录例子说明 AOP
  4. 是一种新的方法论，是对OOP的补充
  5. 主要操作对象时切面，而切面模块化横切关注点
    切面可以简单的理解为就是一个类，横切关注点就是功能
  6. AOP底层就是一个动态代理
  7. AOP好处
  • 代码方便维护和升级
  • 业务模块更简洁，只包含了核心业务代码

4.2 AOP术语

  1. 连接点：类里面哪些方法可以被增强，这些方法称为连接点
  2. 切入点：实际被真正增强的方法称为切入点
  3. 通知（增强）：实际增强的逻辑部分称为通知，且分为以下五种类型：
    1. 前置通知
    2. 后置通知
    3. 环绕通知
    4. 异常通知
    5. 最终通知
  4. 切面：把通知应用到切入点过程

4.3 AspectJ

  1. Spring 框架一般都是基于 AspectJ 实现 AOP 操作，AspectJ 不是 Spring 组成部分，独立 AOP 框架，一般把 AspectJ 和 Spirng 框架一起使 用，进行 AOP 操作
  2. 基于 AspectJ 实现 AOP 操作：
    1. 基于 xml 配置文件实现
    2. 基于注解方式实现（使用）
